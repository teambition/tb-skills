## 认证相关错误

### 错误 1：环境变量未设置

**错误信息**：
```
❌ 错误: 无法获取 User Token
```

**原因**：
- 未设置 `TB_USER_TOKEN` 环境变量
- 未创建 `user-token.json` 配置文件
- 配置文件格式错误

**解决方案**：

**方式 1：设置环境变量**
```bash
export TB_USER_TOKEN="your_user_token_here"
```

**方式 2：创建配置文件**
创建 `user-token.json` 文件：
```json
{
  "userToken": "your_user_token_here"
}
```

**方式 3：Claw 配置**
编辑 `~/.openclaw/openclaw.json`：
```json
{
  "skills": {
    "entries": {
      "dingtalk-tb-ai-skill": {
        "enabled": true,
        "env": {
          "TB_USER_TOKEN": "your_user_token_here"
        }
      }
    }
  }
}
```

### 错误 2：认证失败

**错误信息**：
```
❌ API 调用失败，状态码: 401
```

**原因**：
- User Token 不正确
- User Token 已过期
- User Token 格式错误

**解决方案**：

1. **检查 Token 是否正确**
   - 访问 [Teambition 开放平台](https://open.teambition.com/user-mcp)
   - 重新获取 Token
   - 确保复制完整的 Token 字符串

2. **检查 Token 是否过期**
   - Token 有有效期限制
   - 过期后需要重新创建
   - 建议定期更换 Token

3. **检查 Token 格式**
   - Token 应该是一个长字符串
   - 不要包含多余的空格或换行符
   - 确保没有被截断

## 权限相关错误

### 错误 3：权限不足

**错误信息**：
```
❌ API 调用失败，状态码: 403
```

**原因**：
- User Token 没有相应的权限
- 尝试访问无权限的资源
- 企业管理员限制了权限

**解决方案**：

1. **检查 Token 权限**
   - 访问 [Teambition 开放平台](https://open.teambition.com/user-mcp)
   - 查看 Token 的权限范围
   - 确保有查询任务、项目、成员的权限

2. **联系管理员**
   - 如果权限不足，联系企业管理员
   - 申请相应的权限
   - 或使用有权限的账号创建 Token

3. **检查资源权限**
   - 确保有权访问目标项目
   - 确保有权查看目标任务
   - 私有项目需要是项目成员

## 参数相关错误

### 错误 4：TQL 语法错误

**错误信息**：
```
❌ API 调用失败，状态码: 400
错误信息: TQL syntax error
```

**原因**：
- TQL 语法不正确
- 字段名拼写错误
- 运算符使用错误

**解决方案**：

1. **检查 TQL 语法**
   - 参考 [任务 TQL 参考文档](TQL_REFERENCE.md)
   - 参考 [项目 TQL 参考文档](PROJECT_TQL_REFERENCE.md)
   - 使用正确的字段名和运算符

2. **常见错误示例**

❌ **错误**：
```bash
# 字段名错误
--tql "executor = me()"  # 应该是 executorId

# 运算符错误
--tql "isDone == true"  # 应该是 =

# 引号错误
--tql "content = '测试"  # 缺少闭合引号
```

✅ **正确**：
```bash
--tql "executorId = me()"
--tql "isDone = true"
--tql "content = '测试'"
```

3. **测试 TQL**
   - 先使用简单的 TQL 测试
   - 逐步添加条件
   - 确认每个条件都正确

### 错误 5：参数格式错误

**错误信息**：
```
❌ 参数错误: 无效的项目 ID
```

**原因**：
- ID 格式不正确
- ID 不存在
- 参数类型错误

**解决方案**：

1. **检查 ID 格式**
   - 项目 ID 和任务 ID 应该是 24 位的十六进制字符串
   - 例如：`67ec9b8c3c6130ac88605c3e`
   - 不要包含空格或特殊字符

2. **检查参数类型**
   - `--page-size` 应该是整数
   - `--user-ids` 应该是逗号分隔的字符串
   - 布尔参数不需要值（如 `--no-details`）

## 网络相关错误

### 错误 6：网络连接失败

**错误信息**：
```
❌ 网络错误: Connection timeout
```

**原因**：
- 网络连接不稳定
- 无法访问 Teambition API
- 防火墙阻止了连接

**解决方案**：

1. **检查网络连接**
   - 确保网络连接正常
   - 尝试访问 https://open.teambition.com
   - 检查是否需要代理

2. **检查防火墙设置**
   - 确保防火墙允许访问 Teambition API
   - 检查企业网络策略
   - 必要时联系网络管理员

3. **重试请求**
   - 网络问题可能是临时的
   - 稍后重试
   - 增加超时时间

### 错误 7：请求超时

**错误信息**：
```
❌ 请求超时
```

**原因**：
- 网络速度慢
- 查询数据量过大
- 服务器响应慢

**解决方案**：

1. **减少查询数据量**
   - 使用分页查询
   - 减小 `--page-size` 的值
   - 使用更精确的 TQL 筛选

2. **优化查询**
   - 避免查询所有数据
   - 使用索引字段（如 ID）查询
   - 避免复杂的 TQL 表达式

3. **增加超时时间**
   - 修改脚本中的超时设置
   - 或稍后重试

## 服务器相关错误

### 错误 8：服务器错误

**错误信息**：
```
❌ API 调用失败，状态码: 500
```

**原因**：
- Teambition 服务器内部错误
- 服务暂时不可用
- 数据异常

**解决方案**：

1. **稍后重试**
   - 服务器错误通常是临时的
   - 等待几分钟后重试
   - 避免频繁重试

2. **检查服务状态**
   - 访问 Teambition 官网
   - 查看是否有服务公告
   - 确认服务是否正常

3. **联系技术支持**
   - 如果问题持续存在
   - 记录错误信息和请求 ID
   - 联系 Teambition 技术支持

### 错误 9：服务不可用

**错误信息**：
```
❌ API 调用失败，状态码: 503
```

**原因**：
- 服务维护中
- 服务器过载
- 临时故障

**解决方案**：

1. **等待服务恢复**
   - 查看是否有维护公告
   - 等待服务恢复
   - 避免频繁请求

2. **实现重试机制**
   - 添加自动重试逻辑
   - 使用指数退避策略
   - 设置最大重试次数

## 数据相关错误

### 错误 10：数据不存在

**错误信息**：
```
查询结果为空
```

**原因**：
- 查询条件不匹配任何数据
- 数据已被删除
- 没有权限查看数据

**解决方案**：

1. **检查查询条件**
   - 确认 TQL 条件是否正确
   - 尝试放宽查询条件
   - 检查字段值是否正确

2. **检查数据是否存在**
   - 确认项目/任务是否存在
   - 检查是否被删除或归档
   - 确认有权限查看

3. **调整查询策略**
   - 先查询所有数据
   - 再逐步添加筛选条件
   - 确认每个条件的影响

### 错误 11：数据格式异常

**错误信息**：
```
❌ 数据解析失败
```

**原因**：
- API 返回的数据格式异常
- 数据包含特殊字符
- 编码问题

**解决方案**：

1. **检查 API 响应**
   - 查看原始响应数据
   - 确认数据格式是否正确
   - 检查是否有特殊字符

2. **更新脚本**
   - 确保使用最新版本的脚本
   - 检查是否有已知的兼容性问题
   - 必要时更新依赖包

3. **报告问题**
   - 如果是 API 的问题
   - 记录详细的错误信息
   - 联系技术支持

## 调试技巧

### 1. 查看详细错误信息

大多数脚本会输出详细的错误信息，包括：
- 错误类型
- 错误描述
- 请求 ID（requestId）
- 追踪 ID（traceId）

### 2. 使用简单查询测试

遇到问题时，先使用最简单的查询测试：

```bash
# 测试认证
uv run scripts/get-current-user.py

# 测试项目查询
uv run scripts/query-projects.py --no-details

# 测试任务查询
uv run scripts/query-tasks.py --no-details
```

### 3. 逐步添加条件

从简单到复杂，逐步添加查询条件：

```bash
# 第一步：查询所有任务
uv run scripts/query-tasks.py

# 第二步：添加执行人条件
uv run scripts/query-tasks.py --tql "executorId = me()"

# 第三步：添加状态条件
uv run scripts/query-tasks.py --tql "executorId = me() AND isDone = false"
```

### 4. 检查日志

查看脚本输出的日志信息：
- 请求参数
- API 响应
- 错误堆栈
